## 方式3: 源码安装（开发者）

### 系统要求
- Python 3.11+
- Node.js 18+
- Redis 7.0+

### 快速安装
```bash
# 1. 克隆仓库
git clone https://github.com/gfchfjh/CSBJJWT.git
cd CSBJJWT

# 2. 安装后端依赖
cd backend
pip install -r requirements.txt
playwright install chromium

# 3. 安装前端依赖
cd ../frontend
npm install

# 4. 启动Redis（如果未安装）
# Windows: 使用内置的redis/redis-server.exe
# Linux/macOS: redis-server

# 5. 启动后端
cd ../backend
python -m app.main

# 6. 启动前端（新终端）
cd ../frontend
npm run dev

# 7. 访问应用
http://localhost:9527
```

## 💡 常见问题快速解决

### Q1: KOOK账号显示"离线"？
**v4.1.0智能诊断**：
```
帮助中心 → 故障排查 → 点击"账号离线"诊断
  └─ 自动检测原因并提供解决方案
```

**手动检查**：
1. Cookie是否过期？ → 重新导入Cookie
2. 网络是否正常？ → 检查网络连接
3. 查看详细日志 → 日志页面

### Q2: 环境检查失败？
**v4.1.0一键修复**：
```
向导 → 环境检查 → 点击"一键修复全部"
  ├─ 自动安装Chromium
  ├─ 自动启动Redis
  ├─ 自动安装依赖
  └─ 自动修复权限
```

### Q3: Cookie导入失败？
**v4.1.0智能验证**：
```
系统会自动检测10种常见错误：
  ├─ ✅ 检测到控制台输出 → 提示如何正确导出
  ├─ ✅ 检测到HTML内容 → 提示正确的位置
  ├─ ✅ 检测到格式错误 → 尝试自动修复
  └─ ✅ 提供详细教程链接
```

### Q4: 消息转发失败？
1. 检查Bot配置是否正确
2. 测试Bot连接（Bot配置页面）
3. 查看实时日志了解详情
4. 使用帮助中心的诊断工具

### Q5: 图片转发失败？
**v4.1.0图片策略**：
```
设置 → 图片处理 → 选择策略
  ├─ 智能模式：自动尝试直传→图床→本地备份
  ├─ 直传模式：直接上传到目标平台
  └─ 图床模式：使用内置图床（最稳定）
```
